## Design Context

This project has an impeccable design system captured at the root:

- **[PRODUCT.md](PRODUCT.md)** — strategic layer. Register: **product** · Platform: **web**. Primary user is an on-the-ground ops analyst (bilingual EN/Bangla) reviewing bKash & Nagad Super Agent liquidity/risk.
- **[DESIGN.md](DESIGN.md)** — visual layer. The "Vault" system: institutional teal identity, provider (magenta/orange) vs. severity (red/amber/slate) as strictly separate channels, IBM Plex Sans/Mono + Noto Sans Bengali, flat/border-driven surfaces.

Design principles that govern UI work here: **review, not verdicts** (neutral, non-alarmist — never accuse; the word "fraud" is forbidden in the product); **explainable by construction** (surface the triggering numbers, never a bare score); **show what lives between providers** (provider identity ≠ severity — distinguish by shape, not just color); **earned familiarity** (dense, consistent, calm); **humane precision** (bilingual at parity, careful with people). Read PRODUCT.md and DESIGN.md before designing new screens.
